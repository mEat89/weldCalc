## What changed in this CBFEM-alignment pass

### Stage A — Base-metal solicitation routing (§J4.1 vs §J4.2)
`weldMath.ts::calcBaseMetal` now accepts `solicitation: "shear" | "tension"` and
branches:
- **shear**: `Rn = 0.6·Fy·A` (Eq. J4-3, φ = 1.00) and `Rn = 0.6·Fu·A` (Eq. J4-4, φ = 0.75)
- **tension** (also used for moment-couple demand): `Rn = Fy·A` (Eq. J4-1, φ = 0.90) and `Rn = Fu·A` (Eq. J4-2, φ = 0.75)

Previously every base-metal check used the shear formulas, which inflates the
DCR by 1/0.6 ≈ 1.67× for any tension/moment demand. The hook now passes
`solicitation` through from the UI; tests in `weldMath.test.ts` cover both
branches.

### Stage B — Full §K5 Eq. K5-2/K5-6 Sip moment math
`weldMath.ts` adds `calcSip`, `calcMomentIpCapacity`, and `calcK5GroupCapacity`.
`useHSSCalculation.js` now computes `sipResult`, `momentCap`, and `groupCap`
on every render; the per-face `pFace` for moment is derived from the
extreme-fiber stress (`σ_max = Mu·y_max/Sip`) on the flange face, and the
longitudinal web face gets `P_face = 0` (its bending stress is rolled into
the group `Mn-ip` via the Sip web term `Hb²/(3·sin²θ)`).

The old "share factor SF = L_eff/(L_eff + d/3)" approximation has been removed
from the canonical moment math; `momentShareFactor` is still exposed for
backward-compat trace strings but is not used in any capacity calculation.

### Stage C — Total group capacity card (Check 5)
A new `GroupCapacityCard` component renders after the per-face checks and
shows the connection-level Pn (axial) or Mn-ip (moment) from Eq. K5-1/K5-5
or Eq. K5-2/K5-6. This matches what Hilti CBFEM reports as the headline
"Welds = X%" utilization. Per-face Checks 1-3 remain the default view; the
group card is supplemental.

Same block is appended to the exported PDF report as "Check 5 — Total group
capacity (§K5-1/K5-5)" or "(§K5-2/K5-6)" depending on solicitation.

## Remaining over-conservatism (NOT bugs — intentional)

### Base metal hotspot at K5 Be length

For HSS-to-plate K5: base metal area = t_HSS × Be = 0.116″ × 0.344″ = 0.040 in².
That yields φRn_base ≈ 1.04 kip. Any meaningful demand overstresses it.

Decomposed (report 5, Ls = 0.348″, tension):

```
Flange B (transverse, L_eff = Be = 0.344")
  P_face = 10 × (2.0 / 19.536) = 1.02 kip   (perimeter share = 10.2%)
  Weld metal:  φRn = 2.67 kip → DCR =  38.4%    ← walkthrough cites this
  Base metal:  φRn = 1.04 kip → DCR =  98.3%    ← actual governor
Web H (longitudinal, L_eff = 7.768")
  P_face = 10 × (7.768 / 19.536) = 3.98 kip
  Weld metal:  φRn = 60.20 kip → DCR =   6.6%
  Base metal:  φRn = 23.52 kip → DCR =  16.9%
```

This is the "2D local hotspot vs. 3D stress diffusion" gap the walkthrough
mentions. The math is doing what the codeRef strings say it's doing — and
the conservative-engineering-judgment labels we added make this explicit. A
user reading the trace will see the base metal is the limiter at a deliberately
tight model assumption, not that the weld itself is failing.

### AISC mode is *less* conservative than CBFEM for shear

HSS4x4x1/4 with Vy = 17 kip: Hilti = 49%, ours = 20.3%. We're 29 points lower
than CBFEM because AISC mode treats the plate as rigid (full L_eff = B = 4″),
ignoring the corner stress concentration that CBFEM resolves.

If a user wants to bracket CBFEM, switching to K5 mode tightens L_eff to Be
and brings the calculator into the conservative side.
